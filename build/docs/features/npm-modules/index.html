NPM Modules

Bare Module Imports​

In LiveCodes you can use node-style bare module imports for npm modules like you do in your local development. However, there is no build step required.

e.g. consider the following code:

import { v4 } from 'uuid';

document.body.innerHTML = v4();

If you run it in regular web page, you get this error:

Uncaught TypeError: Failed to resolve module specifier "uuid". Relative references must start with either "/", "./", or "../".

However, in LiveCodes, bare module imports are imported from skypack.dev (which provides ESM versions of NPM packages).

So uuid becomes https://cdn.skypack.dev/uuid;

This is made possible by using import maps.

Demo:

 

You can import React like that:

import React, { useState } from 'react';
import ReactDOM from 'react-dom';

Demo:

It just works without a build step and without you having to worry about. And when you export your code to another service (e.g CodePen or JSFiddle), the full url imports are used, so your code continue to work elsewhere.

It is recommended to use this method for dependencies over using external scripts. The dependencies are explicitly stated in the code. And if you move to a local development environment, your bundler will take care of importing them and doing other optimizations like tree-shaking.

Other CDN Providers​

By default, bare module imports are imported from skypack.dev. You may choose another provider by using a CDN prefix:

uuid → https://cdn.skypack.dev/uuid (info)

skypack:uuid → https://cdn.skypack.dev/uuid (info)

jsdelivr:uuid → https://cdn.jsdelivr.net/npm/uuid (info)

esm.run:uuid → https://esm.run/uuid (info)

esm.sh:uuid → https://esm.sh/uuid (info)

jspm:uuid → https://jspm.dev/uuid (info)

esbuild:uuid → https://esbuild.vercel.app/uuid (info)

unpkg:uuid → https://unpkg.com/uuid?module (info)

github:uuidjs/uuid/main/src/v4.js → https://raw.githack.com/uuidjs/uuid/main/src/v4.js (info)

gitlab:staltz/manyverse/-/blob/master/index.web.js → https://gl.githack.com/staltz/manyverse/-/raw/master/index.web.js (info)

Example:

import React from 'esm.sh:react';

CommonJS Modules​

CommonJS module requires are also supported.

So this also works (although not recommended - use ES6 imports instead):

const { v4 } = require('uuid');

document.body.innerHTML = v4();
